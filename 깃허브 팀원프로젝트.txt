깃허브 프로젝트 활용

관리자가 repository 생성 파일을 올림

각 팀원이 pull로 떙깁니다. git pull git grant

각 팀원이 브랜치를 생성합니다. master branch는 마스터 관리자만 관리하지만 형평성을 위해서 놔둡니다.

작업을 끝났으면 master로 merge병합

다시 팀원이 pull로 땡겨서 현재 작업과 합칩니다.

오류가 나오면 이전 작업으로 돌아가기위해서 git revert를 칩니다.

git clone 주소 git pull

규칙 1개 파일 겹치면 안됩니다. 두명이상 한 파일을 작업을 해서는 안됩니다. 

위키는 계획표 노트와 객체 메서드 용어 설명을 노트하기로 합니다. 공지기능과 같습니다.

issue는 게시판 기능 또는 댓글로써 대화를 할수있도록 도와줍니다.

git branch branchname(branch1)
